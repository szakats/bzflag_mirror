MOTD Version Field Rules


  1. Version "0.0" is a special version that applies to ALL BZFlag versions since MOTD was introduced (1.10).
  2. Before version 2.0.6, BZFlag used a strict version number match; that is, 2.0.2 will match on "2", "2.0," and "2.0.2" only.
  3. Version 2.0.6 and newer in the 2.0 line, and old 2.1s, can use globbing (via bzglob), against the whole app version string ("2.0.8.20060203-MAINT-W32VC7-SDL", for instance).  ? matches any one character, * matches any string.
  4. Recent 2.1 releases and all newer use case-insensitive regexes.  They will simulate the older rules where applicable (except for globbing); any version field that begins with a digit will be matched according to the old rules, so if you want to use regexes starting at the beginning of the string, use ^ to do so explicitly.

Convenient Regex Versioning Templates

  1. Match a single version: ^2\.1\.11\.
  2. Match a version range: ^2\.1\.[7-15]\.
  3. Match a particular platform or group of platforms: -W32VC[78]D?-
  4. Match a particular build: ^2\.1\.11\.20071120-DEVEL-linux-gnu-SDL

Update Notification Triggers

  1. Set the title field to "UPGRADE: BZFlag <version>" (i.e. "UPGRADE: BZFlag 2.0.10")
  2. Set the message field to the URI for the windows installer to enable automatic upgrades.
  3. Set the version number to trigger for the minor version it replaces (i.e. "^2\.1\.[0-9]+" when 2.2.0 is released, "^2\.2\.[0-3]" when 2.2.4 is released, etc).
  4. Note that it is not safe to trigger update notification on non-windows platforms for versions prior to 2.0.11.
